ECMAScript 2015 (ES2015)

  • Also known as ECMAScript 6 (ES6)
  • Specification finalized June 2015
  • Fully-supported in 0 browsers

ECMAScript is the official name of the JavaScript language specification. ECMAScript 5 was finalized 5 years ago. As of this year, ES6 is now officially called ECMAScript 2015. The hope is that ECMAScript 7 will be finalized in 2016. This new language specification was just finalized last month but it is already fully supported in all major browsers. Oh wait, no. I mean it's not supported in **any** browsers. It's going to be a long time before you can confidently use this new language in the browser. However you **can** start using this new language today...

Use next generation JavaScript, today.

You just need to compile your new ECMAScript 6 code down to ECMAScript 5. That way your code will run in all web browsers. With Babel you can do just that.

ES2015 features to use today

  1. template strings
  2. object methods
  3. fat arrow functions
  4. let declarations
  5. for..of
  6. destructuring

I'm going to show you 6 of these features.

Template Strings


var pageNumber = "( " + getCount() + " / " + pageTotal + " )";
            


var pageNumber = `( ${getCount()} / ${pageTotal} )`;

If you have ever tried to use stick a variable inside a string in JavaScript, you'll know it requires concatenation. Look at all those plus signs and quotes. With template strings, you can include your variables right inside your string. Just use backticks and you've made a template string.

Methods


var Duck = Backbone.Model.extend({
  name: function name() {
    return "duck";
  },
  sound: function sound() {
    return "quack";
  }
});
              


var Duck = Backbone.Model.extend({
  name() {
    return "duck";
  },
  sound() {
    return "quack";
  }
});

You probably write a lot of code that look like this. Most JavaScript MVC frameworks include an object syntax that uses keys/value pairs for defining methods and attributes. With the new method syntax, you can remove a lot of the syntax cruft when defining methods.

Fat Arrows


var response = promise.then(function (data) {
  return data.response;
});
              


var response = promise.then(data => data.response);

JavaScript is an event-driven language that loves callbacks. So anonymous functions are very useful in JavaScript. With the new fat arrow syntax, you can define anonymous functions much more elegantly. In this example, we've lost the parenthesis, the curly braces, the return statement, and the word function.

Fat Arrows


var getValue = function () {
  return this.value;
}.bind(this));
              


var getValue = () => this.value;

Fat arrow functions also inherit the "this" and "arguments" bindings from their outer scope. That means you no longer need those "self" or "that" variables you often see used for storing "this".

Let


var array = [];
for (var i = 0; i < 10; i++) {
array.push(Math.random());
}
console.log(i);  // Returns 10
            


let array = [];
for (let i = 0; i < 10; i++) {
array.push(Math.random());
}
console.log(i);  // ReferenceError: i is not defined

Function hoisting, variable hoisting, function-level scoping. It's all gone! Well sort of... If you use "let" instead of "var" to define your variables, they'll use block-level scoping and they will not be hoisted. If you don't know what variable hoisting is, look it up. It's kind of messy. When you start using Babel to transpile your code, the first thing you should do is change all of your code to use "let" instead of "var".

Destructuring Arrays


var numbers = [1, 4, 2];
var x = numbers[0];
var y = numbers[1];
var z = numbers[2];
              


let numbers = [1, 4, 2];
let [x, y, z] = numbers;

Combined with the new "for of" syntax, this code is a lot shorter.

Destructuring Objects


var object = {name: "Blue", value: 5};
var color = object.name;
var value = object.value;
              


let object = {name: "Blue", value: 5};
let {name: color, value} = object;

Combined with the new "for of" syntax, this code is a lot shorter.

Destructuring & For..Of


var coordinates = [{x: 1, y: 4}, {x: 3, y: 3}, {x: 2, y: 2}];
var timesAtHome = 0;

for (var i = 0; i < coordinates.length; i++) {
  var c = coordinates[i];
  if (c.x === 3 && c.y === 3) {
    timesAtHome += 1;
  }
}
              


let coordinates = [{x: 1, y: 4}, {x: 3, y: 3}, {x: 2, y: 2}];
let timesAtHome = 0;

for (let {x, y} of coordinates) {
  if (x === 3 && y === 3) {
    timesAtHome += 1;
  }
}

Combined with the new "for of" syntax, this code is a lot shorter.
